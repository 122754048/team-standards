# 团队规范库 (Team Standards)

> 所有项目必须严格遵守本目录下的规范文档。规范不是建议，是红线。

## 规范目录

| 规范 | 文件 | 适用岗位 |
|---|---|---|
| 产品规范 | [PRODUCT-STANDARD.md](./PRODUCT-STANDARD.md) | PM（小策） |
| 设计规范 | [DESIGN-STANDARD.md](./DESIGN-STANDARD.md) | UI 设计师（小艺） |
| 开发规范 | [DEV-STANDARD.md](./DEV-STANDARD.md) | 客户端开发（小前）、服务端开发（小后） |
| 测试规范 | [TEST-STANDARD.md](./TEST-STANDARD.md) | 测试工程师（小测） |

## 核心原则

1. **禁止空文档** — 每份规范必须包含可执行的操作步骤，不是口号
2. **工具驱动** — 每个岗位必须使用专业工具，不允许纯手工操作
3. **验收闭环** — 每个岗位的输出必须有明确的验收标准
4. **红线不可逾越** — 违反红线的产出物直接打回重做

## 版本管理

- 规范版本号格式：`v1.0.0`（主版本.次版本.修订版本）
- 重大变更需要大哥批准
- 次要变更由 PM 提议，主控审批
- 修订变更（错别字、格式）可由各岗位自行修正
